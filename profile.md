## Projects

### 1) 쇼핑몰(E-commerce) 프로젝트
- 기간: (입력)
- 인원: (입력)
- 역할: 인증/회원, 주문, 위시리스트, i18n, CI/CD, DB 설계 등 기능 구현 참여
- 링크: GitHub (입력) / Demo (입력)

#### 핵심 기능
- **회원 관리**
  - 회원가입/로그인/회원정보 수정/비밀번호 변경/탈퇴(삭제 처리 정책 반영)
  - 이메일 유니크 처리(중복 불가 정책 반영)
- **소셜 로그인**
  - 카카오 로그인 연동(토큰 처리 및 서버 인증 플로우 구성)
- **인증/보안**
  - JWT 기반 Access/Refresh 토큰 전략 적용
  - 토큰 만료/에러 코드 기반 예외 처리 및 재인증 흐름 정리
- **주문**
  - 장바구니 → 주문 → 주문완료 흐름
  - 주문 목록/상세/삭제(정책에 따른 데이터 유지) 처리
  - 주문 완료 페이지 로딩 이슈 방지(세션/상태 복원 방식 적용)
- **위시리스트**
  - 추가/조회/삭제 API 연동
  - 중복 처리 및 첫 클릭 반영 문제 등 UX 이슈 해결
- **다국어(i18n)**
  - Nav 메뉴/카테고리 등 국제화 적용
  - “카테고리는 국제화 없이 진행”된 데이터 구조와의 호환 방식 정리
- **CI/CD**
  - Jenkins 파이프라인 구성 및 배포 실패 원인 추적(JAVA_HOME 등 환경 이슈 대응)
- **DB/설계**
  - Oracle 기반 ERD/다이어그램 산출 및 테이블 관계 정리

#### 트러블슈팅 (문제 → 해결)
- **JWT 인증/필터 예외 처리**
  - 문제: 특정 경로/메서드에서 인증 필터 적용 범위가 꼬이거나 토큰 예외 처리 일관성이 부족
  - 해결: shouldNotFilter 조건 정리 및 에러 응답 규격화(만료/토큰 오류 코드 대응)
- **주문 완료 페이지 무한 로딩/초기 렌더 이슈**
  - 문제: location.state 의존으로 새로고침/직접 진입 시 주문 정보 유실
  - 해결: sessionStorage 기반 복원 로직(“lastOrder”) 추가로 초기 로딩 안정화
- **위시리스트 버튼 첫 클릭 미반영**
  - 문제: API 호출/상태 갱신 타이밍으로 UI가 첫 동작에서 즉시 반영되지 않음
  - 해결: 상태 업데이트 흐름/요청 반환 처리 순서 조정, 목록 재조회 또는 optimistic update 적용
- **Jenkins 배포 실패**
  - 문제: JAVA_HOME 경로 불일치 등 빌드 환경 설정 이슈
  - 해결: Jenkins Tool/JDK 설정 및 서버 환경 변수 점검으로 파이프라인 안정화
